- **Oil price war**: Saudi Arabia vs Russia
- **Liquidity crisis**: Everyone selling everything

## The DeFi Meltdown

**Why This Was Different:**

Previous crashes happened mostly on **centralized exchanges**. This time:

- **$500M+ locked in DeFi** protocols
- **Collateralized debt** across MakerDAO, Compound
- **Smart contracts** executing liquidations automatically
- **No circuit breakers** in DeFi

1. ETH price crashes
2. Collateral values drop
3. Loans become undercollateralized
4. Liquidations trigger automatically

## MakerDAO's Near-Death Experience

**What Is MakerDAO:**

- **Decentralized stablecoin** protocol
- Users lock ETH, borrow DAI (stablecoin)
- Must maintain **150% collateralization**
- If ETH drops too much â†’ liquidated
- Liquidation penalty: 13% + gas fees

**What Went Wrong:**

1. **ETH crashed 56%** in hours
2. **$8M+ positions** needed liquidation
3. **Network congestion**: Gas fees hit 200+ Gwei
4. **Failed liquidations**: Keepers couldn't process
5. **$0 bids won auctions**: System exploited

**The $0 DAI Exploit:**

- **Liquidation auctions** supposed to have bidders
- **Network congestion** prevented competition
- **One bot** submitted $0 bids with high gas
- **Won auctions** for free (~$8M in ETH)
- **MakerDAO insolvent**: $4M+ undercollateralized debt

- **Minted new MKR** to sell for DAI

## What We Learned

**DeFi's Critical Flaws Exposed:**

1. **Gas Dependency**

   - System breaks when network congested
   - Liquidations fail under stress
   - Need Layer 2 or better solutions

2. **Oracle Latency**

   - Price feeds lagged real markets
   - Delayed liquidations = more losses
   - Need faster, more reliable oracles

3. **Liquidity Assumptions**

   - Assumed liquidators always available
   - Network congestion broke this assumption
   - Need backup mechanisms

4. **Collateral Volatility**

   - 150% collateral insufficient in crash
   - Need higher ratios for volatile assets
   - Stablecoin collateral preferred

5. **No Circuit Breakers**
   - Traditional markets pause trading
   - DeFi has no emergency stops (by design)
   - Trade-off: decentralization vs stability

**Immediate Changes:**

1. **Higher collateralization ratios** across DeFi
2. **Better oracle systems** (Chainlink adoption)
3. **Emergency pause functions** added
4. **Minimum bid requirements** in auctions
5. **Layer 2 urgency** increased

**Strategic Shifts:**

- **Multi-collateral DAI**: Diversify beyond ETH
- **USDC as collateral**: Added stable collateral options
- **Protocol insurance**: Nexus Mutual, others
- **Stress testing**: Protocols simulate crashes
- **Liquidity backstops**: Reserve funds

**Philosophical Reckoning:**

- **"Code is law"** met reality (sometimes code fails)
- **Decentralization vs stability** trade-off

- **Governance speed** vs crisis response

## The Comparison to Traditional Finance

**What TradFi Had That DeFi Didn't:**

- **Circuit breakers**: Trading pauses
- **Liquidity injections**: Central bank intervention
- **Margin call extensions**: Human judgment
- **Emergency powers**: Regulators can act
- **Bailouts**: Government backstops

**What DeFi Had That TradFi Didn't:**

- **Transparency**: All liquidations visible
- **Permissionless**: Anyone could help liquidate
- **Global**: 24/7 operation, no holidays
- **Immutable**: Rules enforced by code
- **Composable**: Systems interconnected

**The Irony:**

DeFi's **strengths** (permissionless, automated, always-on) became **weaknesses** during extreme stress.
