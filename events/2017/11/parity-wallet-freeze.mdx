On November 6, 2017, a user accidentally **triggered a critical bug** in Parity's multi-signature wallet library, **freezing 513,774 ETH** (~$280 million at the time, **$1.5+ billion at peak**) across 587 walletsâ€”permanently locking funds with **no way to recover them**, exposing the risks of immutable smart contracts and sparking years of controversy over whether Ethereum should hard fork to save the funds.

## The Incident

**What Happened:**

- **GitHub user "devops199"** (accidentally) triggered vulnerability
- **Parity wallet library** contract self-destructed
- **All wallets** depending on that library instantly **frozen**
- **513,774 ETH locked** forever
- **No private key issue**: Smart contract code bug
- **Irreversible** on immutable blockchain

**The Vulnerability:**

Parity's multi-sig wallet used a **library contract** pattern:

1. **Library contract**: Holds reusable code
2. **Wallet contracts**: Delegate calls to library
3. **Design flaw**: Library could be "initialized" and "killed"

**The Exploit:**

```solidity
function initWallet(...) {
    // Should only be callable once
    // But library contract wasn't initialized
}

function kill() {
    // Self-destruct function
    // Should only be callable by owner
}
```

**The Accident:**

1. User **"devops199"** claimed ownership of library (wasn't initialized)
2. Then **called kill()** function
3. Library contract **self-destructed**
4. All dependent wallets **broke instantly**
5. Funds **frozen forever**

## The Technical Details

**The Library Pattern:**

Smart contracts can use **libraries** to save gas:

```solidity
contract Wallet {
    address library;

    function execute() {
        library.delegatecall(...); // Use library code
    }
}
```

**The Problem:**

Parity's library **itself was a contract** with state:

```solidity
contract WalletLibrary {
    address owner;

    function initWallet() {
        owner = msg.sender; // Initialize owner
    }

    function kill() {
        require(msg.sender == owner);
        selfdestruct(owner); // BOOM
    }
}
```

**The Bug:**

1. Library never initialized (owner = 0x000)
2. Anyone could call `initWallet()` and become owner
3. Then call `kill()` to destroy library
4. All wallets using library **break forever**

**The Fix:**

Modern libraries use **library keyword**:

```solidity
library SafeLibrary {
    // Cannot have state
    // Cannot be initialized or killed
    // Pure code only
}
```
